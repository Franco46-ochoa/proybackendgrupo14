Reglas de ramas
  • Nunca pushear directo a main o develop. Solo mediante PR.
  • Ejecutar npm run lint y npm run build antes de cada push.
  • Nombre de commit en español imperativo: "agregar endpoint POST /api/auth/login".
  • Al mergear a main: tag semántico (v1.0.0-sprint1, v2.0.0-final).
  • Pricila revisa PRs de frontend. Franco revisa PRs de backend. Nadie es cuello de botella único.
  • 30 min diarios de code review cruzado rotativo (los 5 en videollamada). Cada uno explica lo que hizo y cómo. Esto prepara la defensa individual.
